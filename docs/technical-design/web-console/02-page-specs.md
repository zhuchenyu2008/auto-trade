# 02. Web 控制台页面规格

## 1. 页面分层

V1 页面按实现优先级分为两层：

- P0：必须先定稿并优先接入真实数据
- P1：在核心工作台稳定后再补齐

### 1.1 已登录页面统一骨架

除登录页外，所有页面统一运行在“固定顶栏 + 固定左侧栏 + 中央工作区”的控制台骨架内：

- 顶栏只承载全局状态、高优先级风险提示和少量全局动作
- 左侧栏承载一级导航，不因页面不同而频繁改变结构
- 中央工作区承载当前页面主任务，优先使用分区、表格、列表和抽屉组织内容

页面规格只定义各页面在主工作区中的职责与区块，不重复发明新的页面总骨架。

### 1.2 页面气质基线

各页面在实现时继续遵循以下表现约束：

- 少圆角，少大阴影，少漂浮卡片
- 优先通过标题、分隔线、表头、状态条和对齐关系建立秩序
- 首屏优先呈现风险、状态、待办和操作入口，不做展示型 hero
- 可靠感优先于亲和感，避免过软、过亮、过度装饰

| 页面 | 优先级 | 主要目的 |
| --- | --- | --- |
| 登录页 | P0 | 完成单用户认证入口 |
| 总览页 | P0 | 提供系统总状态和待处理事项 |
| 频道页 | P0 | 管理监听来源和抓取健康 |
| 日志页 | P0 | 排查处理链路和异常 |
| 人工确认页 | P0 | 处理新开仓低置信度确认 |
| 真实订单页 | P1 | 查看真实订单、成交和持仓 |
| 虚拟持仓页 | P1 | 查看频道级解释层状态和 PnL |
| 系统设置页 | P1 | 查看和修改运行参数 |

## 2. 登录页

### 2.1 页面目标

让使用者快速进入系统，同时明确当前这是一个受保护的交易控制台。

### 2.2 页面区块

- 品牌或系统名称区
- 登录表单区
- 安全提示区

### 2.3 交互要求

- 采用 `6` 位密码输入
- 支持登录中状态
- 支持失败提示
- 支持多次失败后的临时锁定提示

### 2.4 状态要求

- 未登录时只能访问本页
- 会话失效时应回到本页并提示原因
- 不展示任何敏感配置内容

## 3. 总览页

### 3.1 页面目标

让使用者在最短时间内判断：

- 当前是否在安全环境中
- 当前系统是否健康
- 当前有没有必须马上处理的事情

### 3.2 页面区块

- 顶部风险状态条
- 系统摘要区
- 频道健康摘要区
- 待确认摘要区
- 最近异常与最近动作区

### 3.3 核心信息

- 当前环境
- 全局交易开关
- 系统健康状态
- 待确认数量
- 各频道最近抓取状态
- 最近失败任务或警告

### 3.4 主要动作

- 进入人工确认页
- 进入日志页
- 进入频道页
- 打开环境切换或全局开关确认流程

### 3.5 视觉与布局要求

- 总览页首屏应更像“状态台 + 待办入口”，而不是展示型首页
- 顶部风险状态条必须稳定可见，并与全局顶栏形成清晰分层
- 摘要信息允许分区呈现，但不应堆成夸张的卡片墙
- 若存在待确认项、异常或环境风险，应在桌面端首屏直接可见

## 4. 频道页

### 4.1 页面目标

管理监听对象，并快速看出哪个频道有问题。

### 4.2 页面区块

- 页面标题与操作区
- 频道表格区
- 新增或编辑频道抽屉
- 频道详情抽屉

### 4.3 表格字段

- 频道名称
- 频道来源标识
- 启停状态
- 最近抓取时间
- 最近成功时间
- 最近错误摘要
- 最近消息处理结果

### 4.4 主要动作

- 新增频道
- 编辑频道
- 启用频道
- 停用频道
- 查看最近状态

### 4.5 关键规则

- 删除动作默认采用逻辑删除或等效停用策略
- 停用频道不清空历史消息、历史决策和历史虚拟持仓
- 表格中应能一眼区分“抓取异常”和“仅暂无新消息”

## 5. 日志页

### 5.1 页面目标

成为单条业务链路的主排查入口。

### 5.2 页面区块

- 筛选栏
- 实时日志列表
- 链路详情抽屉

### 5.3 筛选维度

- 时间范围
- 日志级别
- 模块
- 频道
- 环境
- `correlation_id`

### 5.4 列表字段

- 时间
- 级别
- 模块
- 事件摘要
- 频道
- 环境
- `correlation_id`

### 5.5 交互要求

- 新日志应支持实时追加
- 用户在查看历史时，不应被强制跳回列表底部
- 点击某条日志后，应能查看完整上下文与关联对象

## 6. 人工确认页

### 6.1 页面目标

让使用者对低置信度新开仓动作做出快速且可审计的判断。

### 6.2 页面布局

推荐使用“左侧队列 + 右侧详情”的双栏结构。

左侧队列展示：

- 频道
- 标的
- 动作类型
- 置信度
- 创建时间
- 当前状态

右侧详情展示：

- 原始消息
- 相关上下文摘要
- AI 决策结果
- 关键价格参数
- 当前环境
- 确认按钮
- 拒绝按钮

### 6.3 交互要求

- 列表切换时详情区局部更新
- 已失效确认项应明确标红或标灰
- 确认前再次校验当前环境和执行有效性

### 6.4 关键规则

- 同一确认项只能成功执行一次
- 只有新开仓低置信度动作进入该页
- 已有持仓的跟进动作不应进入该确认页

## 7. 真实订单页

### 7.1 页面目标

查看交易所真实执行结果，而不是解释层推断。

### 7.2 页面区块

- 订单表格
- 成交表格
- 当前真实持仓摘要
- 订单详情抽屉

### 7.3 关键字段

- 标的
- 方向
- 订单状态
- 委托价格
- 成交数量
- 环境
- 来源决策或关联 ID

### 7.4 关键规则

- 明确标记该页是交易所事实视图
- 与虚拟持仓页的入口和标题不能混淆

## 8. 虚拟持仓页

### 8.1 页面目标

查看频道级交易叙事、虚拟仓位状态和频道级 PnL。

### 8.2 页面区块

- 虚拟持仓列表
- 频道级 PnL 摘要
- 持仓生命周期详情抽屉
- 与真实订单映射区

### 8.3 关键字段

- 频道名称
- 标的
- 方向
- 虚拟数量
- 虚拟均价
- 当前状态
- 已实现 PnL
- 未实现 PnL

### 8.4 关键规则

- 必须显式标注“虚拟”字样
- 需要能回看其关联消息、决策和真实订单映射

## 9. 系统设置页

### 9.1 页面目标

管理运行参数与风控边界，而不是展示历史明细。

### 9.2 页面分组

- 运行环境组
- 模型与思考等级组
- 交易默认参数组
- 人工确认阈值组
- 新开仓资金比例范围组
- 危险操作区

### 9.3 主要动作

- 修改模型
- 修改思考等级
- 修改默认杠杆
- 修改确认阈值
- 修改资金比例范围
- 切换环境
- 开关全局交易

### 9.4 关键规则

- 敏感凭据不在此页明文展示
- 高影响配置变更需要二次确认
- 配置变更后需明确提示何时生效

## 10. 跨页面共用交互

V1 建议统一实现以下共用交互，而不是每页各自发明一套：

- 全局高风险确认模态框
- 详情抽屉
- 状态标签体系
- 空状态模板
- 错误重试提示
- `correlation_id` 快速跳转入口

## 11. 页面规格验收要点

- P0 页面都能在假数据阶段独立演示完整主路径
- 总览、日志、人工确认之间能通过对象链接互相跳转
- 真实订单页与虚拟持仓页视觉和文案明确区分
- 任一高风险动作都不会直接在首击后执行
