# 03. Web 控制台视觉与交互系统

## 1. 视觉主张

### 1.1 Visual Thesis

V1 Web 控制台的视觉方向定义为：

“像一套可靠的交易控制台，而不是一张柔软、圆润、带氛围感的个人仪表盘。”

这意味着界面应该具备以下气质：

- 稳定
- 克制
- 偏专业
- 高密度但可扫描
- 让人信任，而不是让人先感受到装饰感

### 1.2 Content Plan

控制台的视觉层级按以下顺序组织：

1. 顶部风险与环境信息
2. 左侧稳定导航
3. 当前页面主工作区
4. 次级上下文与详情

任何页面都不应让装饰性元素压过真实业务状态。

### 1.3 Interaction Thesis

V1 建议只保留少量但明确的动效：

- 顶栏与主工作区在进入页面时快速分层出现
- 日志或实时状态更新时使用轻量高亮提示新增
- 抽屉和确认模态使用短促、稳重的进入与退出动画

动效目标不是营造“高级感表演”，而是让层级和状态变化更清楚。

## 2. 视觉关键词

本轮界面关键词明确为：

- 可靠感
- 控制感
- 稳定感
- 专业感

它们对应的反面也要明确避免：

- 过软
- 过圆
- 过亮
- 过度装饰
- 过度“作品集化”

## 3. 布局语言

### 3.1 骨架结构

主骨架采用：

- 顶栏
- 侧边栏
- 主工作区

这套结构在桌面端应尽量稳定，不因为进入不同页面就大幅改变位置关系。

### 3.2 结构表达方式

可靠感主要通过以下元素建立：

- 清晰边界线
- 稳定栅格
- 统一表头与分区标题
- 适度对比的表格和列表
- 固定位置的状态条与导航

而不是通过以下方式建立：

- 大面积模糊玻璃层
- 很重的投影
- 夸张渐变背景
- 大量浮动卡片

### 3.3 圆角策略

本轮已确认整体要“少一些圆角”，因此圆角策略建议如下：

- 顶栏和侧边栏：`0-8px`
- 普通面板：`8-12px`
- 表格、输入框、标签：`6-10px`
- 模态框：`10-14px`

不再采用首页大圆角卡片、超大胶囊按钮和过于柔和的包裹轮廓。

## 4. 基础设计 token

V1 建议以中性浅色工作台为主，降低暖色氛围和玻璃感，保留后续暗色主题扩展位。首轮 token 可按以下方向定义：

```css
:root {
  --bg-canvas: #f3f5f7;
  --bg-topbar: #111821;
  --bg-sidebar: #18212b;
  --bg-surface: #ffffff;
  --bg-subtle: #f7f9fb;
  --line-soft: #d7dee7;
  --line-strong: #b7c1cd;
  --text-strong: #18212b;
  --text-body: #344054;
  --text-muted: #667085;
  --accent-info: #175cd3;
  --accent-success: #067647;
  --accent-warning: #b54708;
  --accent-danger: #b42318;
  --accent-live: #8b1e1e;
}
```

设计倾向说明：

- 颜色更中性
- 对比更稳定
- 顶栏与侧栏更像控制台基础设施，而不是装饰层
- 阴影降低，边线加强

## 5. 字体与数字系统

建议只使用两套字体：

- 中文与界面正文：`MiSans`、`HarmonyOS Sans SC` 或同级别清晰无衬线字体
- 数字与代码类信息：`JetBrains Mono`

规则如下：

- 页面标题、区块标题、正文和辅助文字要有清楚的字号梯度
- 价格、仓位、PnL、时间、ID 等高频扫描信息优先使用等宽数字
- 不使用营销式超大标题和浮夸字重
- 标题更偏功能性，不偏海报感

## 6. 布局与间距

建议采用 `8px` 基础间距体系，并对以下层级保持统一：

- 页面外边距
- 区块间距
- 表格行高
- 抽屉内信息分组间距

布局原则如下：

- 页面以“分区 + 表格 + 列表”代替“套娃卡片”
- 表格是主信息承载器之一，应优先保证密度和可扫描性
- 抽屉内容按“摘要在上，证据在下，操作在底”组织
- 首页优先是状态板，而不是大视觉展示页

## 7. 组件气质规范

### 7.1 顶部状态栏

- 必须固定在全局顶部
- 应与主内容形成明显分层
- 背景更稳重，支持强状态展示
- 实盘状态应有更强烈的视觉信号

### 7.2 左侧导航

- 作为长期稳定存在的一级导航
- 不做大块发光和大胶囊按钮
- 激活态要清楚，但不花哨
- 导航项应更像专业控制台条目，而不是营销站点菜单

### 7.3 表格

- 允许较高信息密度
- 通过对齐、字重和行内标签增强扫描效率
- 使用清楚分隔线，不依赖厚阴影
- 优先减少无意义卡片包裹

### 7.4 状态标签

状态标签应形成稳定语义：

- 中性状态：描边或低饱和底色
- 正常状态：低饱和绿色系
- 警告状态：低饱和橙色系
- 危险状态：更高对比的红色系

标签形状建议更克制，避免大圆角胶囊化过强。

### 7.5 抽屉

- 用于承载详情与上下文
- 头部展示对象标题和关键摘要
- 底部展示主操作按钮
- 轮廓清晰，边界明确

### 7.6 模态框

- 只用于强打断
- 默认不用于普通详情查看
- 高风险动作模态框需要二次文案确认
- 轮廓比普通面板更强，但不必过圆

## 8. 动效规范

V1 动效应满足以下要求：

- 明显但克制
- 快速
- 可在移动端保持流畅
- 只服务于层级、反馈和风险提示

推荐首批保留以下三类动效：

- 页面切换时轻量淡入
- 抽屉滑入滑出
- 新日志或状态变更的短暂高亮

不建议在首轮加入复杂视差、长时间滚动动画或大面积装饰性过渡。

## 9. 响应式与可访问性

### 9.1 响应式原则

- 桌面端优先保证顶栏 + 侧栏 + 主工作区的完整结构
- 平板端允许侧栏折叠
- 手机端优先保证查看与确认，不强行保留全部并排信息

### 9.2 可访问性原则

- 颜色不是唯一编码方式
- 关键操作按钮保持足够点击面积
- 键盘焦点状态清楚可见
- 日志流和状态更新不要造成阅读跳跃

## 10. 明确禁止项

首轮视觉设计应避免以下失败模式：

- 首页卡片马赛克
- 大面积玻璃拟态
- 很重的圆角和软糯按钮
- 风险色到处滥用
- 真实状态和虚拟状态只有颜色差异、没有结构差异
- 大量厚重阴影替代结构线
- 把日常操作台做成宣传型 landing page
- 为了“科技感”加入无意义渐变和发光

## 11. 视觉系统验收要点

- 首页在隐藏所有装饰后，仍然层级清楚
- 使用者在三秒内能看出当前是否实盘、是否有待确认项、是否有异常
- 顶栏、侧栏和主工作区的主次一眼可辨
- 真实订单区与虚拟持仓区即使放在同一页，也不会被误读为同一种数据
- 所有高风险动作都比普通按钮更显著，但不至于整页失控
- 圆角、阴影、渐变被明显收敛后，页面仍然成立
