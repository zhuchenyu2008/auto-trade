# 03. Web 控制台视觉与交互系统

## 1. 视觉主张

### 1.1 Visual Thesis

V1 Web 控制台的视觉方向定义为：

“像一张克制的交易编辑台，而不是一套堆满卡片的通用后台模板。”

这意味着界面应该具备以下气质：

- 冷静
- 克制
- 高密度但不拥挤
- 风险感明确
- 有质感但不过度装饰

### 1.2 Content Plan

控制台的视觉层级按以下顺序组织：

1. 顶部风险与环境信息
2. 当前页面核心工作区
3. 次级说明与历史明细
4. 高风险确认入口

任何页面都不应让装饰性元素压过真实业务状态。

### 1.3 Interaction Thesis

V1 建议只保留少量但明确的动效：

- 首次进入应用时，顶部状态栏与主工作区分层进入
- 日志或实时状态更新时，使用轻量高亮提示新增，而不是整页闪烁
- 抽屉打开与关闭使用有重量感的水平滑入，强调“进入上下文”

## 2. 视觉原则

### 2.1 不做后台模板式卡片拼盘

- 首页不用大面积统计卡片墙
- 常规数据优先使用布局、标题、表格和分区，而不是一堆独立卡片
- 卡片只用于确实需要包裹单一交互的对象

### 2.2 真实状态与虚拟状态必须视觉分层

- 真实交易所状态使用更强的标题与更稳重的标签样式
- 虚拟子仓位使用明确的“虚拟”标识和次一级层级
- 两者的分层不能只依赖颜色，必须同时依赖标题、标签、分区和文案

### 2.3 高风险颜色只留给高风险信息

- 实盘
- 强平
- 手动开仓
- 严重错误

这些场景才允许使用高饱和警示色，避免警示色在普通状态中被滥用。

## 3. 基础设计 token

V1 建议先以浅色主题为主工作台，保留后续暗色主题扩展位。首轮 token 可以按如下方向定义：

```css
:root {
  --bg-canvas: #f4f0e8;
  --bg-surface: #fbf8f2;
  --bg-elevated: #ffffff;
  --bg-rail: #ebe3d6;
  --line-soft: #d7cfbf;
  --line-strong: #b8ae99;
  --text-strong: #1f2630;
  --text-body: #394150;
  --text-muted: #6a7280;
  --accent-info: #175cd3;
  --accent-success: #0f766e;
  --accent-warning: #b54708;
  --accent-danger: #b42318;
  --accent-live: #9b1c1c;
}
```

这些值不是最终视觉稿，但应作为原型阶段的稳定基线，避免后续页面各自长成不同风格。

## 4. 字体与数字系统

建议只使用两套字体：

- 中文与界面正文：`MiSans`、`HarmonyOS Sans SC` 或同级别清晰无衬线字体
- 数字与代码类信息：`JetBrains Mono`

规则如下：

- 页面标题、区块标题、正文和辅助文字要有清楚的字号梯度
- 价格、仓位、PnL、时间、ID 等高频扫描信息优先使用等宽数字
- 不使用营销式超大标题和浮夸字重

## 5. 布局与间距

建议采用 `8px` 基础间距体系，并对以下层级保持统一：

- 页面外边距
- 区块间距
- 表格行高
- 抽屉内信息分组间距

布局原则如下：

- 页面以“分区”代替“套娃卡片”
- 表格是主信息承载器之一，应优先保证密度和可扫描性
- 抽屉内容按“摘要在上，证据在下，操作在底”组织

## 6. 组件气质规范

### 6.1 顶部状态栏

- 必须固定在全局顶部
- 背景与正文区域明确区分
- 实盘状态应有更强烈的视觉信号

### 6.2 表格

- 允许较高信息密度
- 通过对齐、字重和行内标签增强扫描效率
- 避免对每个单元格加粗边框

### 6.3 状态标签

状态标签应形成稳定语义：

- 中性状态：描边或低饱和底色
- 正常状态：低饱和绿色系
- 警告状态：低饱和橙色系
- 危险状态：更高对比的红色系

### 6.4 抽屉

- 用于承载详情与上下文
- 头部展示对象标题和关键摘要
- 底部展示主操作按钮

### 6.5 模态框

- 只用于强打断
- 默认不用于普通详情查看
- 高风险动作模态框需要二次文案确认

## 7. 动效规范

V1 动效应满足以下要求：

- 明显但克制
- 快速
- 可在移动端保持流畅
- 只服务于层级、反馈和风险提示

推荐首批保留以下三类动效：

- 页面首屏分层淡入
- 抽屉滑入滑出
- 新日志或状态变更的短暂高亮

不建议在首轮加入复杂视差、长时间滚动动画或大面积装饰性过渡。

## 8. 响应式与可访问性

### 8.1 响应式原则

- 桌面端优先保证高密度、多列表和抽屉操作
- 平板端允许部分区块折叠
- 手机端优先保证查看与确认，不强行保留全部并排信息

### 8.2 可访问性原则

- 颜色不是唯一编码方式
- 关键操作按钮保持足够点击面积
- 键盘焦点状态清楚可见
- 日志流和状态更新不要造成阅读跳跃

## 9. 明确禁止项

首轮视觉设计应避免以下失败模式：

- 首页卡片马赛克
- 风险色到处滥用
- 真实状态和虚拟状态只有颜色差异、没有结构差异
- 大量厚边框和重阴影
- 把日常操作台做成宣传型 landing page
- 为了“科技感”加入无意义渐变和发光

## 10. 视觉系统验收要点

- 首页在隐藏所有装饰后，仍然层级清楚
- 使用者在三秒内能看出当前是否实盘、是否有待确认项、是否有异常
- 真实订单区与虚拟持仓区即使放在同一页，也不会被误读为同一种数据
- 所有高风险动作都比普通按钮更显著，但不至于整页失控
