# 02. 后端能力分块与主流程

## 1. 目标与范围

本文件用于把 Phase 2 的“要交付什么”拆成可落地能力块和主流程，避免实现时边界漂移。

生效范围仅限：

- 认证与会话
- 运行时设置
- 日志查询与实时流
- 健康检查与依赖状态
- 前后端最小联调闭环

## 2. 能力分块总览

| 能力块 | 主要职责 | 关键输入 | 关键输出 |
| --- | --- | --- | --- |
| `auth-core` | 登录、会话、登出、锁定状态判定 | 密码、会话 cookie、登录行为 | 会话事实、认证状态 |
| `security-guard` | 限流、失败计数、临时锁定 | 登录请求、IP、会话 | 锁定结果、限流结果 |
| `runtime-settings` | 运行参数读取和写入 | 设置请求、环境上下文 | 设置快照、变更审计 |
| `logs-query` | 日志列表查询与筛选 | 日志过滤参数 | 日志列表和分页游标 |
| `events-stream` | SSE 推送日志与状态 | 事件总线输入 | 实时事件流 |
| `health-service` | 健康检查与依赖状态评估 | DB/Redis 连通性 | `live`/`ready`/`deps` 状态 |

## 3. 认证能力流程

登录主流程：

1. 接收 `6` 位密码输入。
2. 检查当前锁定状态和限流状态。
3. 使用强哈希比对密码。
4. 登录成功则创建会话并重置失败计数。
5. 登录失败则递增失败计数并在阈值后触发临时锁定。
6. 记录审计与日志（成功、失败、锁定、解锁）。

会话主流程：

1. 请求进入后解析 `HttpOnly` cookie。
2. 校验会话签名和过期时间。
3. 通过则注入 `current_session`。
4. 失效则返回 `UNAUTHORIZED`。

登出主流程：

1. 校验会话存在。
2. 使会话失效（服务端撤销或版本失效）。
3. 清除 cookie。
4. 写入审计日志。

## 4. 设置能力流程

读取流程：

1. 校验会话。
2. 读取当前有效配置快照。
3. 返回前端可见字段。

写入流程：

1. 校验会话和输入 schema。
2. 校验风险边界（枚举值、区间、环境约束）。
3. 在单事务内写入新配置版本或更新记录。
4. 生成审计记录并广播设置变更事件（可选）。
5. 返回最新设置快照。

Phase 2 写入范围建议控制为：

- 模型名
- 思考等级
- 默认杠杆
- 新开仓资金比例区间
- 上下文窗口长度
- 全局交易开关

说明：

- 环境切换属于高风险动作，允许在 Phase 2 预留接口位，但应要求二次确认语义和审计字段。

## 5. 日志与实时流流程

日志查询流程：

1. 校验会话。
2. 解析过滤条件（级别、模块、环境、频道、`correlation_id`）。
3. 返回列表和分页信息。

SSE 流程：

1. 建立连接时验证会话。
2. 发送 `system.heartbeat` 初始事件。
3. 推送 `log.created` 和 `system.status.changed`。
4. 定期发送心跳保活。
5. 连接异常时由客户端重连。

日志与 SSE 字段必须保持一致，至少包含：

- `timestamp`
- `level`
- `module`
- `environment`
- `message`
- `correlation_id`

## 6. 健康检查流程

`/health/live`：

- 仅返回进程状态。

`/health/ready`：

- 并发探测 PostgreSQL 与 Redis。
- 任一依赖失败则返回不可就绪。

`/health/deps`：

- 返回依赖细项状态和最近探测时间。

## 7. 最小联调闭环

Phase 2 应至少跑通以下闭环：

1. 前端登录成功后可拿到会话事实。
2. 前端可读取并更新运行时设置。
3. 前端日志页可拉取日志列表。
4. 前端可接入 SSE 并看到实时日志或心跳。
5. 前端顶栏可读取健康状态并正确展示。

## 8. 异常路径与处理要求

必须显式处理的异常路径：

- 密码错误
- 会话过期
- 登录失败触发锁定
- 设置写入参数非法
- 设置写入冲突（并发）
- 日志查询参数非法
- SSE 连接鉴权失败
- DB 或 Redis 不可用

异常处理规则：

- 前端可消费的错误码稳定。
- 返回 `request_id` 便于排障。
- 严禁“写失败但返回成功”。

## 9. 交付分解建议

建议按如下顺序拆任务：

1. `health-service` 与配置加载。
2. `auth-core` 与 `security-guard`。
3. `runtime-settings`。
4. `logs-query`。
5. `events-stream`。
6. 前后端联调与回归测试。
