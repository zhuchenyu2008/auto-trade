# 自动交易系统分工测试清单（AI + 人工）

> 更新时间：2026-03-22（AI 测试已执行：Phase 0、Phase 3.1~3.3）
> 对应主流程清单：`docs/project-phase-checklist.md`
> 使用规则：每个步骤完成后，先做 AI 测试，再做人工测试；两边都通过，才把主流程步骤标记为完成。

## 1. 通用测试规则

- [ ] 规则 1：AI 测试未通过，不进入人工测试。
- [ ] 规则 2：人工测试发现业务偏差，回到开发修复后重新走 AI + 人工双测。
- [ ] 规则 3：每个步骤都必须保留测试证据（命令输出、截图、日志、`correlation_id`）。
- [ ] 规则 4：高风险动作（实盘切换、强平、手动开仓、核心风控变更）必须额外做二次确认测试。
